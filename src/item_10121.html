I'm skeptical about J#.

I've been quite surprised at the way Microsoft is positioning their four main .NET languages.  The surprising part is that there are four languages instead of three. 

These three languages all make sense to me:

  • VB.NET (for people who don't want to wear out the semicolon on their keyboard)
  • C++ (for people who pride themselves on using a language that few people really understand)
  • C# (for people like me who really just want a Java clone)

But the fourth language doesn't seem to fit into this crowd.  Does J# have a place in the long term?  Several times recently, Microsoft's articles have described J# as if it were a full-fledged peer of its three siblings.  But each time I see this, I find myself wanting to stand and applaud a really fine acting job.  Who at Microsoft really believes J# is here to stay?

I've got two reasons for my skepticism about the long term future of J#:

First reason:  Does anybody in Redmond actually eat this dogfood? 

Microsoft has a dismal track record of maintaining and improving developer tools it does not actually use for its own products.  I have a strong suspicion that nobody at Microsoft is actually using J#, except for all the interns writing samples for it. 

People would find it much easier to put their trust in J# if we started hearing stories about how Microsoft is depending on J# for their next product.

Second reason:  Porting Java to C# is just too easy.

The server for SourceOffSite Classic was written in Java, actually Visual J++ 6, running on the Microsoft JVM.  Unfortunately all support for this JVM has ceased, so it looks like we need to move.

We tried porting to J#, although I'll admit we didn't try very hard.  If the port to J# had been completely painless and free of effort, we would probably be on J# now.  But it didn't Just Work.

So we tossed J# and ported to C#, which turned out to be really easy.  Whole sections of code compiled and worked with no change at all.

I think Microsoft has a positioning clash here.  C# has the position that J# wants.  Both languages are being perceived as "the Microsoft replacement for Java".  The market will not let both languages have this position -- one must lose.  Suspicions about Microsoft's long-term committment to J# will be the deciding factor, and C# will be the one left standing in the end.
