--- layout: default esbma_id: 1270 date: 2004-07-23 14:44:31 keywords: (cornsharp) (daypage) ---  Java vs. C# (Tuesday, 18 March 2003)  

I ran across an interesting tale which attempts to explain how Java Swing became so lame.  This explains a lot.  Java is a technology that I really wanted to like.  Unfortunately our attempts to use Java for GUI work were very disappointing.

(Edited:  I was later told the above tale was a hoax.  The link now seems broken as well.)

On Windows, C# (with Windows Forms) is what Java should have been.  We used it to create SourceGear Vault, and the result was an app which feels completely native on Windows.  I'd obviously like to see this technology become cross-platform, but for now, .NET is a great way to write real apps for Windows.
